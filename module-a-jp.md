# テストプロジェクト概要 - モジュールA - フロントエンド

## 競技時間

競技者はモジュールAを完了するために **3時間** 与えられます。

## はじめに

モジュールAでは、バックエンドや外部サービスを使わない**シングルページのフロントエンドゲーム**を実装します。このゲームは、高度なIT学習プラットフォームである **SkillShare Academy** 向けにブランド化された、クラシックなSnakeゲームのバージョンです。

ゲームは、任意の **JavaScriptフレームワーク** または **Vanilla JavaScript** を使用して実装してください。

## プロジェクトとタスクの概要

あなたのタスクは **"The Developer's Learning Path"** を実装することです。これはSnakeスタイルのゲームで、プレイヤーは固定グリッド上で **Student** を操作し、**IT Skills** を集めて **Knowledge Base** を成長させながら、**Burnout** と **Spaghetti Code** を避けます。

### 用語

- **Student**: 蛇の頭。プレイヤーキャラクターを表します。
- **Knowledge Base**: 蛇の胴体（または尾）。プレイヤーがIT Skillsを収集することで成長します。
- **IT Skills**: プレイヤーが収集するアイテム。Knowledge Baseを成長させ、スコアを増やします。
- **Burnout**: Studentが壁に衝突したときのゲームオーバー条件。
- **Spaghetti Code**: Studentが自身のKnowledge Baseに衝突したときのゲームオーバー条件。
- **Heads-Up Display**: 盤面上部に表示される3つの統計情報。

### デザインガイドライン

assetsにはゲームの**サンプル動画**が含まれています。動画のデザインをそのままコピーしてはいけません。ワイヤーフレームとして捉え、レイアウトと要素のみを参考にし、style-guideに従いつつ創造的で独自性のあるデザインにしてください。

- 示されたレイアウト構造とコンポーネント配置に従ってください。
- **SkillShare Academy** のビジュアル方向性を適用してください。assetsフォルダ内のstyle-guideに従い、**ダーク系**の色を使用してください。
- レスポンシブレイアウトを確保してください。一般的な解像度（フルHDを含む）で、盤面と操作UIが不適切に切り取られることなく使用可能である必要があります。体験全体は、必須の縦スクロールなしで収まる必要があります。小さいビューポートでもレイアウトが破綻しないように視覚的に適応させてください。ただし、タッチスワイプ操作は必須ではありません。
- モダンなUI/UX実践を用いてください。キーボードのみで操作するユーザーにとって、インタラクティブ要素のコントラストとフォーカス状態が適切である必要があります。

### 画面/ビュー

以下の画面を実装する必要があります。

#### スタート画面

ページ読み込み後、最初のプレイ開始前に表示されます。この画面はゲームボード上のモーダルまたはオーバーレイとして実装して構いません。

このビューには以下を含めてください。

- **タイトル**: **"The Developer's Learning Path"** と **SkillShare Academy** に言及すること。
- **簡単な説明**: 移動方法（**矢印キー** / **WASD**）、**一時停止**方法（**Space**）、および **IT Skills** が何を表すか。
- 主要ボタン **"Start Learning Path"**: 新しいランを開始し、スコアをリセット、初期 **Student** と **Knowledge Base** を配置、**IT Skill** を1つ生成し、ゲームループを開始すること。

#### ゲームボード画面

ゲームが**プレイ中**または**一時停止中**のメイン状態です。

このビューには以下を含めてください。

- 20×20セルのゲーム**グリッド**。
- グリッド内の **Student**、**Knowledge Base**、現在の **IT Skill** が明確に区別できること。**IT Skills** はassetsフォルダで提供されるアイコンを使用して表示してください。
- **Scoring** セクションで定義された3つのスコア（**Certificates earned**、**Developer level**、**Best run**）を表示する **HUD（Heads-Up Display）**。HUDの算出方法が分かるよう、UI上でユーザーフレンドリーに示してください。

**一時停止中**は、ゲームが一時停止していることを説明し、**Resume** を提供する**モーダルまたはオーバーレイ**を表示してください。一時停止中はゲームが進行してはいけません。

#### ゲームオーバー画面

ランが **Burnout** または **Spaghetti Code** により終了したときに表示されます。この画面はゲームボード上のモーダルまたはオーバーレイとして実装して構いません。

このビューには以下を含めてください。

- **死亡原因**を明示すること: **Burnout**（壁への衝突）と **Spaghetti Code**（自己衝突）を区別し、ゲーム文脈での意味を短く説明すること。
- **最終スコア**: ラン終了時点の **Certificates earned** と **Developer level**。
- 主要ボタン **"Start new learning path"**: スタート画面と同じルールで即座に新しいランを開始すること。

### スコアリング

ゲーム中、HUDには以下3つのスコアを表示する必要があります。

1. **Certificates earned**: 現在のランのスコア（現在のランで収集した **IT Skills** の数）。
2. **Developer level**: 現在の **Certificates earned** から計算されるレベル値。計算式は `level = 1 + floor(certificates / 2)`（certificateを2つ獲得するごとに1レベル上昇、開始レベルは1）。
3. **Best run**: 個人最高の **certificates** スコア。ページ再読み込み後も保持されること。

## ゲームメカニクスと実装要件

ゲームは以下のメカニクスとルールに従う必要があります。

- **Grid**: プレイフィールドは **20×20** のグリッドです。各セルは空、または **Student**、**Knowledge Base** のセグメント、**IT Skill** のいずれかで占有されます（同時に1セル1要素のみ）。
- **Initial game state**: 各ラン開始時、**Student** はグリッド中央から右向きで開始し、**Knowledge Base** の長さは2、さらにランダムな空セルにランダムな **IT Skill** を1つ生成します。
- **Movement and game speed**: ゲームがアクティブな間（未一時停止かつゲームオーバーでない間）、**Student** は現在方向に自動で移動します（上・下・左・右のみ）。プレイヤーはキーボード操作（矢印キーとWASD）で **Student** の方向を変えられますが、蛇は自動前進します。スキルを多く集めるほど移動速度は上がる必要がありますが、プレイ可能性を保つ妥当な速さであること（例: 制御不能なほど速くしない）。
- **Illegal reversal（不正な反転）**: プレイヤーは即座に逆方向へ向きを変えてはいけません。例えば、蛇が右に進んでいる場合、少なくとも別方向への1回の移動を挟むまで左には変更できないこと。
- **Keyboard controls**: 方向変更は **矢印キー** と **WASD**、**Space** で**一時停止**と**再開**。
- **IT Skills**: **Student** が **IT Skill** のあるセルに入ると、そのスキルは消費され、スコアが増加し、**Knowledge Base** が1セグメント成長し、新しいランダムな **IT Skill** がランダムな空セルに出現します。利用可能なスキルはassetsフォルダに用意されています。
- **Collisions**:
  - **Burnout**: **Student** がグリッド外へ移動した場合（壁衝突）→ **Game Over**、理由は **Burnout**。
  - **Spaghetti Code**: **Student** がすでに **Knowledge Base** に占有されたセルへ移動した場合（自己衝突）→ **Game Over**、理由は **Spaghetti Code**。
- **HUD**: トップバーの3つの統計（**Certificates earned**、**Developer level**、**Best run**）は **Scoring** セクションと一致している必要があります。
- **Best run**: ベストラン値（単一ランでの最高 **certificates**）はページ再読み込み後も**永続化**され、アプリ読み込み時に復元される必要があります。
- **Paused state**: **一時停止中**は、ゲームがいかなる形でも進行してはいけません。プレイヤーは進行状況やスコアを失うことなく再開できる必要があります。

### 動きとアニメーション

ゲームイベントに紐づくアニメーションを含め、操作に意図が感じられるようにしてください。  
**例**（デザインに合う任意の一部を実装）:

- **IT Skill** を収集したときの**パルス**または**スケール**アニメーション
- ランが **Burnout** で終了したときの**シェイク**、**フラッシュ**、または**ボーダーパルス**
- **Pause**、**Game Over**、**Start** オーバーレイの**滑らかなフェードイン** / **opacity** / **backdrop** トランジション
- フォーカス時またはホバー時のコントロールの**トランジション**。

すべてのアニメーションを実装する必要はありませんが、ゲームは動的で、プレイヤー操作に対して反応が良いことが求められます。少なくとも1種類のアニメーションを実装してください。

モジュールAはGoogle Chromeで評価されます。評価には、機能動作、ビジュアルデザイン、アニメーション、プレイアビリティ、コード品質が含まれます。
