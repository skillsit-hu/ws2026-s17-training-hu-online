# 测试项目大纲 - 模块 B - 速度测试

## 比赛时长

参赛者有 **1 小时** 完成模块 B。

## 简介

本模块要求你完成一组彼此独立的任务。这些任务用于测试你对 HTML、CSS、JavaScript 以及基础算法的掌握情况。

- 以 A 开头的任务要求你使用 HTML 和 CSS，不使用 JavaScript。
- 对于以 B 开头的任务，你可以（并且必须）使用 JavaScript。
- 对于 C 类任务，你需要使用 JavaScript 以及基础数据结构与算法知识。

## 项目结构

你可以使用 `vanilla-js-app` 模板创建你的仓库。然后，为每个任务创建 9 个文件夹：`a1`、`a2` 等。在这些文件夹中放置 `index.html` 文件作为你的解答。同时请将任务所需的媒体资源放在对应任务的子文件夹中。部署后，我们应能通过 `/a1`、`/a2` 等路径访问你的解答。

你的 `/c1`、`/c2` 和 `/c3` 文件夹中应仅包含 `index.js` 文件。我们会在本地运行它们。

---

## 任务

### A1: CSS 心形 (简单)

使用 CSS 创建一个心形。

### A2: Mac 任务栏放大动画 (中等)

仅使用 HTML 和 CSS，实现 Apple MacBook 任务栏放大动画。请参考示例视频，以更好地理解任务要求。

我们已提供部分初始样式，你无需为这些内容担心。

### A3: 3D 魔方 (困难)

魔方是一种 3D 组合配色谜题，由匈牙利建筑师 Ernő Rubik 于 1974 年发明。请仅使用 HTML 和 CSS 创建一个 3D 魔方。魔方应在屏幕中垂直和水平居中，并像示例视频中那样持续旋转。

魔方各面的颜色如下：

- 顶面：**白色**
- 底面：**黄色**
- 前面：**红色**
- 后面：**橙色**
- 左面：**绿色**
- 右面：**蓝色**

### B1: 画中画 (简单)

在 assets 中给定了一个视频。请将该视频显示在页面上。初始状态下视频应为暂停状态。同时显示一个 "Toggle Picture in Picture" 按钮。  
点击该按钮后，视频应以画中画模式播放。这意味着视频会在屏幕右下角、位于其他元素上方播放。用户还可以将这个小窗口拖动到屏幕任意位置。即使切换到其他浏览器标签页或应用程序，视频仍应可见。  
再次点击该按钮应退出画中画模式。

我们将使用最新版本的 Google Chrome 测试你的解答。

### B2: 跟随光标 (中等)

在屏幕中央放置一个圆。该圆应为白色背景，并带有黑色 1px 边框。

从圆中引出一条线。线的一端始终连接在圆心，另一端跟随用户光标。也就是说，用户光标与圆心始终由这条线连接。

请参考示例视频。

### B3: 绘图工具 (困难)

创建一个 300px\*300px 的 canvas，并使其在屏幕中居中。用户应能在 canvas 上用黑色进行绘制。canvas 应有一个轻微边框，便于用户识别绘图区域。

在 canvas 下方创建一个导出按钮。点击该按钮后，应将绘制结果导出并下载为名为 `drawing.png` 的 PNG 图片。按钮不需要额外样式。

请注意，导出的图片应具有白色背景，以确保用户绘制内容清晰可见。

### C1: 热门课程 (简单)

给定一组课程报名数据。请实现函数 `getTopCourses(enrollments, topN)`，根据报名人数返回前 N 门课程。函数应返回课程名称与报名人数组成的数组，并按报名人数降序排列。

请查看 [/assets/module-b/c1/index.js](/assets/module-b/c1/index.js) 文件。

### C2: 数组分块 (中等)

实现函数 `chunkArray(array, chunkSize)`，该函数接收一个数组和分块大小作为参数，并返回一个新数组，其中原数组会按指定分块大小拆分为多个子数组。如果原数组元素总数不能被分块大小整除，则最后一个分块可以包含更少元素。

请查看 [/assets/module-b/c2/index.js](/assets/module-b/c2/index.js) 文件。

### C3: 两数之和 (困难)

你会得到一个**数字**列表（可能包含负数）以及一个**目标值**。

请在列表中找到**两个不同位置**，使得这两个位置上的值**相加等于目标值**。按 `[i, j]` 形式返回这两个位置，且满足 `i < j`。

你可以假设**恰好存在一组**正确答案。

请查看 [/assets/module-b/c3/index.js](/assets/module-b/c3/index.js) 文件。
