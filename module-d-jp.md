# テストプロジェクト概要 - モジュール D - LMS フロントエンド（学習サイト）

## 競技時間

競技者はこのモジュールを完了するために **3 時間**与えられます。

## はじめに

モジュール D は、**SkillShare Academy コンテンツサービス**用のフロントエンドを実装することに重点を置きます。これは独立した **LMS（学習）サイト**であり、登録済みの学習者がコースを開き、章を順番に進め、リッチなレッスン内容を読み、章末のクイズを完了できるようにします。

## プロジェクトとタスクの概要

SkillShare Academy（SSA）は、アカウント、受講登録、クレジット、章の完了を扱う**メインバックエンド**と、コースメタデータ、章のコンテンツブロック、メディア、クイズを保存して提供する**コンテンツサービス**を使用します。**モジュール D** は、**コンテンツサービス**と通信する**ブラウザアプリケーション**のみです。

コンテンツサービスとメインバックエンドは提供されています。LMS は、学習者がメインの SkillShare Academy プラットフォーム（ダッシュボード）にサインインしたときに受け取った**同じ Bearer token**を使用しなければなりません。通常、ダッシュボードは**クエリ文字列内の token** を付けて LMS を開きます。LMS フロントエンドはそれを保存し、以後の API 呼び出しで使用します。**Token の有効期間：** メインバックエンドが発行する token は、ログイン後 **60 秒**で期限切れになります。その後、ユーザーがダッシュボードで再度サインインするまでは `401 Unauthorized` が返ることを想定してください。

### ダッシュボードと LMS の URL

評価と開発のために、完全に動作する **SSA ダッシュボード**アプリケーションが提供されています。デプロイ先は次のとおりです。

`https://cXX-YYYY-dashboard.ssa.skillsit.hu`

（競技用の `cXX` ユーザー名と `YYYY` PIN を使用してください。）

**テスト：** すべてのシードユーザーは、開発とテストのためにダッシュボードへサインインする際、パスワード `password123` を使用します（例：**[alice@example.com](mailto:alice@example.com)** / **password123**）。

**長期有効テスト token（約 7 日）：** 開発中にダッシュボードログイン token の **60 秒**の期限切れを避けるため、**[alice@example.com](mailto:alice@example.com)** 用のこの Bearer token（モジュール C と同じ）を使用できます。LMS URL に `?token=` と token を追加するか、アプリが想定する方法（例：`localStorage`）で保存してください。

```text
eyJhbGciOiJIUzI1NiIsInR5cCI6IlNTQSJ9.eyJzdWIiOiIxIiwiZXhwIjoxNzc4NjUxNjI5fQ.SupdoOSna89QLZejnwZMHTUqsS7lPC_OWvY3Q7_RXIU
```

**ヒント：** 開発と UI テストでは、**Tailwind CSS & ShadCN UI Tutorial** コースを **`/courses/tailwind-css-shadcn-ui-tutorial`** で開くことを推奨します。このコースはシードデータ内で最も豊富なレッスン内容（見出し、段落、リスト、画像、動画、リンク、クイズ）を持っているため、すべてのコンテンツブロックとレイアウト規則を確認するのに最適です。**[alice@example.com](mailto:alice@example.com)** がまだそのコースに登録されていない場合は、ダッシュボードで登録してください。

デフォルトでは、ダッシュボードから LMS への**すべてのナビゲーション**は、競技者が**デプロイした** LMS フロントエンドを指します。

`https://cXX-YYYY-module-d.ssa.skillsit.hu`

**ローカル開発：** **同じダッシュボード**から**ローカルで実行中**の LMS（例：Vite の `http://localhost:5173`）を使用するには、ブラウザの **localStorage**（ダッシュボードの origin 上）にある **`lmsSiteUrl`** エントリを、開発用 LMS のベース URL（末尾のスラッシュなし）に設定してください。ダッシュボードはこの値を使用するため、「open course / continue learning」リンクは、デフォルトのデプロイ済み LMS ではなく開発サイトへ移動します。デフォルトの動作に戻すには、`lmsSiteUrl` を削除またはリセットしてください。

**モジュール C との関係：** モジュール C は**コンテンツサービス** API とデータモデルを定義し、実装します。モジュール D は、必要な LMS のすべての動作（コース詳細、章コンテンツ、クイズ検証）について**その API のみ**を利用します。メインバックエンドは引き続きアカウントと完了記録を処理しますが、**LMS フロントエンドはメインバックエンドの HTTP API を呼び出してはなりません**。必要な場合（例：クイズに合格したとき）は、コンテンツサービスがメインバックエンドと連携します。

**コンテンツサービスのベース URL：** `https://cXX-YYYY-module-c-solution.ssa.skillsit.hu`。ここで `cXX` はユーザー名、`YYYY` は PIN コードです（正確なパターンはモジュール C の handout を参照）。

**API ドキュメント：** コンテンツサービスの OpenAPI 仕様は `assets` ディレクトリにあります：[`/assets/module-c/api/ssa-content-service-openapi.yaml`](/assets/module-c/api/ssa-content-service-openapi.yaml)。これは LMS が呼び出す必要のあるすべてのエンドポイントのリファレンスです。

**データベース：** 競技者はコンテンツサービスのデータベースへ直接アクセスできません。スキーマと動作は API ドキュメントで定義されます。

フロントエンドは JavaScript **フレームワーク**を使用して実装しなければなりません。アプリケーションは**シングルページアプリケーション（SPA）**でなければなりません。**ルーティングはフレームワークで処理しなければなりません。** ディープリンクされた URL をリロードした場合、保存されていないユーザー入力を除き、アプリ内でその場所へ移動したときと同じビューを表示する必要があります（認証状態がストレージから復元された後）。

## 要件

このフロントエンドの目的は、学習者がコンテンツサービス API を通じてコース教材を学習し、クイズを完了できる明確なインターフェースを提供することです。

フロントエンドは次の機能要件を実装しなければなりません。

- 関連する HTTP ステータスコードのエラー処理（[エラー処理](#エラー処理) を参照）
- コンテンツサービスが **`401 Unauthorized`** を返した場合（loader、クイズ送信など、どのリクエストでも）、ユーザーが再度サインインできるように**ブラウザをメイン SSA ダッシュボードへリダイレクト**すること。[エラー処理](#エラー処理) の `401` ルールを参照
- リロードをまたいで token を保持するためのローカルストレージ
- 順番に基づく章アクセスと完了状態を持つコースページ
- 必要なすべてのコンテンツブロックタイプと章クイズをレンダリングする章ページ

### デザインガイドライン

handout には、コースページと章ページの**両方**について、**SkillShare Academy（SSA）デザインガイド**と**グラフィックデザイン素材**（参照画像）が含まれています。競技者はデザインガイドに従い、提供されたビジュアルに**できる限り近づけて**実装する必要があります（レイアウト、階層、余白、指定された色とタイポグラフィ、コンポーネントの扱い）。

デザインガイドが特定の詳細（例：特定のコンテンツブロックやエラー状態）を扱っていない場合は、同じパターンを拡張して UI の一貫性を保ってください。実装は、下記の章ページで説明されている動画レイアウトを含め、画面サイズに応じて**レスポンシブ**でなければなりません。

### エラー処理

コンテンツサービスはさまざまなエラーを返す可能性があります。フロントエンドはそれらを処理し、**理解しやすい**メッセージを表示しなければなりません（生のスタックトレースは表示しないこと）。少なくとも次を処理してください。

- `400 Bad Request` - リクエストが不正です。ユーザーに簡潔に知らせてください。
- `401 Unauthorized` - token が欠落、無効、または期限切れです。**ユーザーをメイン SSA ダッシュボードアプリケーションへリダイレクト**してください（[ダッシュボードと LMS の URL](#ダッシュボードと-lms-の-url) のダッシュボードベース URL、例：`https://cXX-YYYY-dashboard.ssa.skillsit.hu` を使用します。通常はアプリのルート、または handout で指定された入口点です）。ユーザーが LMS を離れて再認証できるよう、**フルページナビゲーション**（`window.location` または同等の方法）が必要です。また、処理の前または処理の一部として、保存済みの LMS token を消去してください。インラインエラーだけを表示してユーザーを LMS に残してはいけません。
- `403 Forbidden` - ユーザーはそのリソースへアクセスできません（例：章がロックされている）。その章はまだ利用できないことを説明してください。
- `404 Not Found` - コースまたは章が見つかりません。コンテンツが利用できないことをユーザーに知らせてください。
- `500 Internal Server Error` - サーバーエラーです。一時的な問題が発生していることをユーザーに知らせてください。

クイズ検証は、アプリケーション固有の payload を返す場合があります（例：クイズには合格したが、メインプラットフォームに章の完了を記録できなかった）。UI は OpenAPI / モジュール C ドキュメントに記載されたとおりに、これらのケースを表示しなければなりません。

### ページ

次のビューを実装しなければなりません（正確なパス名は参照実装に従っても構いません）。

#### コースページ（`/courses/:slug`）

`GET /api/courses/:slug` を `Authorization: Bearer <token>` 付きで呼び出し、コース詳細を読み込みます。

ページには次を含める必要があります。

- **コースヘッダー：** タイトル、説明、難易度、総章数、総クレジット（API が返す値）。
- **Hero / 画像：** SSA bucket から hero 画像を取得します：`https://bucket.ssa.skillsit.hu/<slug>.webp`（`<slug>` を API から返されたコース slug に置き換えます）。
- **章リスト：** すべての章を順番に表示し、それぞれにタイトル、短い説明、クレジット、**完了済み**状態を含めます。
- **順番に基づくアクセス：** 最初の章、または**直前の**章が完了している章だけが**アンロック**されます（章へのリンク）。ロックされた章は明確に示し、章ビューへ**遷移してはなりません**。
- **完了済みの章**は、視覚的に区別できる必要があります（例：バッジやスタイル）。

#### 章ページ（`/courses/:slug/chapters/:chapterId`）

`GET /api/courses/:slug/chapters/:chapterId` で章コンテンツを読み込みます（Bearer が必要）。

**`content` 配列（レッスン本文）：** JSON body は単一の**章 payload**です（外側の `chapter` オブジェクトでラップされていません）：[`/assets/module-c/api/ssa-content-service-openapi.yaml`](/assets/module-c/api/ssa-content-service-openapi.yaml) の schema `ChapterContentResponse` を参照してください。メタデータ（`courseId`、`chapterId`、`title`、任意の `description`、`credits`）と `quiz` に加えて、**`content`** フィールドは**順序付きの学習ブロック一覧**です。

- 各項目は **`type`** によって識別されるオブジェクトで、章内の表示順を定義する **`orderIndex`** を持ちます（昇順でソート）。
- 許可される **`type`** 値は、保存されているコンテンツブロックに対応します：**`h1`**、**`h2`**、**`h3`**、**`h4`**、**`paragraph`**、**`list_item`**、**`image`**、**`video`**、**`link`**。
- **見出し**（`h1`-`h4`）は、見出しテキストを **`text`** で公開します。
- **`paragraph`** と **`list_item`**：サービスがサーバー側でリッチテキストを組み立て、クライアントは埋め込み可能な **`html`** を受け取ります。**`rawText`** はソース表現として存在する場合があります。表示には **`html`** を使用してください。**`list_item`** では、各配列要素が通常 1 行のリスト項目です。**`html`** は通常 `<li>…</li>` のようなフラグメントです。下記のレンダリング規則を参照してください。
- **`image`**：**`url`** と **`alt`**。
- **`video`**：**`url`** と **`title`**（title は空の場合があります）。
- **`link`**：**`url`** と **`title`**（アンカーのラベル）。

同じレスポンス内の **`quiz`** オブジェクトには、章末の多肢選択問題が含まれます。これは **`content`** とは別です。

ページには次を含める必要があります。

- **スティッキー章バー：** 学習者がレッスンをスクロールしている間も表示され続ける**スティッキー**なサブヘッダーを実装します。**1 行**の中に、(1) 親コースページ（`/courses/:slug`）へ戻る **Back to course** リンクと、(2) 目次ドロワーを開く**ボタン**を含めてください。TOC コントロールをグローバルサイトヘッダーだけに置いては**いけません**。これはスティッキー章バー内で **Back to course** の横に配置されるものです。ビジュアル参照：[`chapter-page-light-with-toc.png`](/assets/module-d/design/chapter-page-light-with-toc.png)、[`chapter-page-dark-with-toc.png`](/assets/module-d/design/chapter-page-dark-with-toc.png)。
- **目次（ドロワー）：** ボタンは **Contents** というタイトルの**ドロワー**（側面からスライドアウトするパネル）を開きます。章の **`content`** 配列から項目を生成します。**`type`** が **`h1`**、**`h2`**、または **`h3`** のすべてのブロックを、**`orderIndex`** の昇順で含め、各ブロックの **`text`** を使用します。**見出し階層**を表示してください（例：**`h2`** を **`h1`** の下にインデントし、**`h3`** を **`h2`** の下にインデントする）。各項目は、レッスン内の該当見出しへスクロールする、またはフォーカスを移動する**ナビゲーションコントロール**でなければなりません。**`h4`** 見出しは本文にはレンダリングされますが、TOC には**含めません**。
- **ヘッダー：** 章タイトル、任意の説明、クレジット、および API レスポンスから適切なコース/章識別子（参照レイアウトのように、スティッキー章バーの下に配置）。
- **レッスンコンテンツ：** `content` 配列を **`orderIndex`** 順にレンダリングします。対応するブロックタイプ：
  - **見出し** `h1`-`h4`：API のプレーンテキストを使用します。
  - **段落：** サーバーが提供する HTML をレンダリングし、読みやすい本文スタイルを適用します。
  - **リスト項目：** 連続する `list_item` 行は、**単一の** `<ul>` にまとめなければなりません。各項目の `html` は通常 1 つの `<li>…</li>` です。
  - **画像：** `url` と `alt` を持つ `<img>`。`alt` が存在する場合はキャプションとして使用します。
  - **動画：** `url` を `src` とする `<video controls playsInline>`。**title** が存在する場合、**デスクトップ**幅（例：約 900px 以上）では、プレイヤーを行の約 **2/3**、タイトル（および任意のラベル）を約 **1/3** に配置します。小さい画面では、動画を上、テキストを下に**縦積み**します。
  - **リンク：** 外部リンクには `rel="nofollow noopener noreferrer"` と `target="_blank"` を使用します。

- **クイズ：** 章に問題が定義されている場合：
  - 各問題と**ラジオ**選択肢を表示します（各問題につき 1 つの選択）。
  - **すべて**の問題が回答された場合にのみ**送信**できるようにします。
  - OpenAPI で指定されたとおり、回答を `.../quiz/validate` へ `POST` します。
  - 合格、不合格、エラー、特殊ケース（例：クイズには合格したが完了同期に失敗した）について、明確なフィードバックを表示します。
  - 完了に成功した後は、コースページに更新後の完了状態が反映されるように、コース/章の状態を**更新**します（例：再取得またはルーターの再検証）。

#### レイアウトとテーマ

- **Header：** ロゴまたはタイトルは**ダッシュボード**ホーム URL へリンクします。**ダッシュボードのコース**へのリンクも含めます（[ダッシュボードと LMS の URL](#ダッシュボードと-lms-の-url) と同じダッシュボードベース URL、例：`https://cXX-YYYY-dashboard.ssa.skillsit.hu` を使用）。ダーク/ライトテーマの切り替えは `localStorage` に保存します。
- **テーマ切り替え：** アプリケーションは**ダークテーマ**（デフォルト）と**ライトテーマ**の両方をサポートしなければなりません。両テーマは SSA スタイルガイド（`assets/style-guide/ssa-style-guide.md`）で定義されています。現在のテーマは `localStorage` に永続化し、ページリロード後にユーザーの設定が復元されるようにしてください。ページを開くときにテーマカラーのちらつきが発生してはいけません。
- token が存在する場合、ユーザーが**サインイン済み**であることを表示します（例："Signed in"）。メインバックエンドからユーザーの**名前**を読み込むことは**任意**です。
- **読み込み中：** ナビゲーション中または遅いリクエスト中には、見える形の読み込み状態を表示します。

## 評価

モジュール D は、提供されたバージョンの **Google Chrome** を使用して評価されます。評価には、**機能**の動作と**ユーザー体験**（明確さ、レスポンシブ対応、エラー処理）が含まれます。

**重要：** 競技者は、OpenAPI で指定された**ドキュメント化済みのコンテンツサービス**エンドポイントのみを使用しなければなりません。提供されたバックエンドやデータベースの変更は、フロントエンド評価の範囲外です。
